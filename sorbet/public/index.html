It works!

Sorbet is now up and running, but you missed to configure it properly.

Prepare a sorbet.profile file. You can copy/paste the contents of this template and modify the folder paths accordingly to your setup. 

{
	"port":  8080,
	"hosts": {
		"localhost":   "/var/www/website_dev",
		"example.org": "/var/www/website"
	}
}

Now restart Sorbet with the created profile. If this installation is portable, replace sorbet with the path to ./bin/sorbet.sh. 

sorbet stop;                          # stop this instance
sorbet start /path/to/sorbet.profile; # start a new instance
